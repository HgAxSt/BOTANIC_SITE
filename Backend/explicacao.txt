1. Express — O "Gerente" do seu Backend

Pense no Node.js como uma cozinha industrial. O Express organiza essa cozinha:
- define os balcões (rotas),
- escolhe os caminhos para as requisições,
- aponta qual função prepara cada "prato".
Ex.: a rota /usuarios chama a função que busca usuários.

2. Mongoose — O "Tradutor Bilíngue e Organizador"

Seu código fala JavaScript; o banco (MongoDB) fala "Mongo". O Mongoose traduz entre eles.
Além disso, impõe um molde (Schema) para os dados, evitando bagunça no banco.
Ex.: todo usuário precisa ter nome e e‑mail.

3. CORS — O "Porteiro" do seu Prédio

Navegadores bloqueiam requisições entre origens diferentes por segurança.
O CORS é a lista de autorização que o servidor entrega ao porteiro (navegador),
permitindo que, por exemplo, localhost:5500 converse com localhost:3000.

4. Dotenv — O "Cofre de Senhas"

Nunca deixe senhas no código (nem envie para o GitHub). Guarde segredos em .env.
O Dotenv carrega essas variáveis no momento da execução, mantendo-as fora do repositório.
